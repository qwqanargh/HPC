
## Медианный фильтр с использованием CUDA и Texture Memory

Этот проект реализует 9-точечный медианный фильтр с использованием CUDA для ускорения обработки изображений на GPU, что позволяет значительно сократить время обработки по сравнению с традиционными CPU-реализациями. Для доступа к пикселям изображения используется Texture Memory через современный Texture Object API.
Ниже описаны функции и распараллеленные операции.

### Зависимости
Для данной реализации требуются следующие пакеты Python:
- `PIL` (Pillow) для загрузки и сохранения изображений в формате BMP.
- `pycuda` для программирования CUDA на Python.
- `numpy` для числовых операций.
- `opencv-python` для чтения изображений и преобразования в оттенки серого.
- `matplotlib` для отображения изображений до и после фильтрации.
- `ctypes` для вызова CUDA Runtime API (`libcudart.so`) напрямую из Python.

### Обзор кода

#### 1. Импорт библиотек
- **PIL**: Используется для загрузки входного изображения и сохранения результата в BMP.
- **pycuda**: Используется для управления ресурсами GPU и выполнения CUDA-ядра.
- **numpy**: Используется для числовых операций, особенно для подготовки данных изображения.
- **cv2**: Используется для чтения BMP-файла и преобразования цветного изображения в grayscale.
- **matplotlib**: Используется для отображения изображений до и после фильтрации.
- **ctypes**: Используется для прямого вызова `cudaCreateTextureObject()` и `cudaDestroyTextureObject()` из `libcudart.so`.

#### 2. Добавление шума «соль и перец»
- **add_salt_pepper_noise**: Функция добавляет случайный шум к изображению с заданной вероятностью (по умолчанию 10%):
  - **Генерирует матрицу случайных чисел** того же размера, что и изображение.
  - **Устанавливает пиксели в 0** («перец») там, где случайное число меньше `prob / 2`.
  - **Устанавливает пиксели в 255** («соль») там, где случайное число в диапазоне `[prob / 2, prob)`.
  - **Сохраняет зашумлённое изображение** в `input_image.bmp` для последующей передачи фильтру.

#### 3. CUDA-ядро для медианного фильтра
- **CUDA-ядро** `median_filter_texture` выполняется на GPU и:
  - **Определяет позицию каждого пикселя** на основе индексов потоков (`threadIdx`) и блоков (`blockIdx`).
  - **Читает 9 соседних пикселей** через `tex2D<unsigned char>()` из Texture Object, обеспечивая кэшированный доступ к данным.
  - **Обрабатывает граничные пиксели** — координаты зажимаются через `min/max`, отсутствующие соседи берутся из ближайшего пикселя.
  - **Сортирует 9 элементов** методом пузырька для нахождения медианы.
  - **Присваивает значение медианы** (элемент с индексом 4) выходному изображению.

#### 4. Создание Texture Object
- **`cudaResourceDesc`**: описывает источник данных — `cudaArray` с 8-битными беззнаковыми значениями (grayscale).
- **`cudaTextureDesc`**: задаёт параметры — режим адресации `CLAMP`, фильтрация `POINT`, ненормализованные координаты.
- **`cudaCreateTextureObject()`**: создаёт texture object и возвращает 64-битный handle, передаваемый в ядро как `unsigned long long`.
- **`cudaDestroyTextureObject()`**: освобождает ресурс после завершения обработки.

### Распараллеливание
Вся обработка изображения распараллелена, так как каждый поток обрабатывает один пиксель одновременно. Это критично для повышения производительности, особенно на больших изображениях:
- **Сетка и блоки**: Изображение делится на блоки 16×16 потоков, позволяя множеству потоков работать над различными пикселями одновременно. Количество блоков вычисляется как `ceil(width / 16) × ceil(height / 16)`. Каждый поток выполняет ядро независимо, работая над отдельным пикселем.

#### 5. Функция для применения медианного фильтра
- **apply_median_filter_texture**: Основная функция выполняет следующие действия:
  - Загружает входное изображение и преобразует его в grayscale (`uint8`).
  - Копирует данные в `cudaArray` через `cuda.Memcpy2D` для размещения в texture memory.
  - Создаёт Texture Object через `create_texture_object()` с помощью вызовов `libcudart`.
  - Выделяет выходной буфер в глобальной памяти GPU (`cuda.mem_alloc`).
  - Определяет блоки и сетки для выполнения CUDA-ядра.
  - Вызывает CUDA-ядро, передавая handle texture object и размеры изображения.
  - Измеряет время обработки через **CUDA Events** с синхронизацией `stop_evt.synchronize()`.
  - Копирует результат обратно на хост и сохраняет выходное изображение в BMP.
  - Освобождает Texture Object вызовом `cudaDestroyTextureObject()`.

### Заключение
Эта реализация демонстрирует возможности использования CUDA и Texture Memory для задач обработки изображений, таких как медианная фильтрация, используя распараллеливание для достижения более быстрых временных показателей обработки. Способность GPU обрабатывать множество пикселей одновременно значительно сокращает время, необходимое для выполнения операций над большими изображениями, что делает его подходящим для задач обработки изображений в реальном времени.
