# Умножение матриц на CPU и GPU

Этот проект реализует умножение матриц двумя способами:
- Последовательное умножение на **CPU**
- Параллельное умножение на **GPU** с использованием технологии CUDA

---

## Описание функций

### `matrix_multiply_cpu(A, B)`

Функция выполняет умножение двух матриц на **CPU**.
Алгоритм использует тройной цикл для вычисления каждого элемента результирующей матрицы.

- **Параметры:**
  - `A`: Первая матрица (numpy array, float32)
  - `B`: Вторая матрица (numpy array, float32)
- **Возвращает:**
  - `answer`: Результат умножения матриц
  - `elapsed_time`: Время, затраченное на выполнение операции (в секундах)

#### Детали реализации

Операция умножения матриц выполняется через тройной цикл по индексам `i`, `j` и `k`.  
Каждый элемент результирующей матрицы вычисляется как скалярное произведение соответствующей строки матрицы `A` и столбца матрицы `B`.  
Сложность алгоритма — O(N³), что делает его крайне медленным для больших матриц.

---

### `matrix_multiply_gpu(A, B)`

Функция для умножения двух матриц на **GPU** с использованием библиотеки PyCUDA.  
Вычисление выполняется параллельно, что даёт значительный выигрыш по времени.

> **Важно:** CUDA-ядро компилируется один раз при загрузке модуля, а не при каждом вызове функции. Замеряется только время вычисления (выделение памяти, передача данных и запуск ядра), но не компиляция.

- **Параметры:**
  - `A`: Первая входная матрица (numpy array, float32)
  - `B`: Вторая входная матрица (numpy array, float32)
- **Возвращает:**
  - `C`: Результат умножения матриц (numpy array, float32)
  - `elapsed_time`: Время выполнения операции (в секундах)

#### Детали реализации

- **Проверка размеров:** В начале функции проверяется совместимость матриц для перемножения; при несоответствии выбрасывается `ValueError`.
- **Выделение памяти на GPU:** Для входных и выходной матриц (`A_gpu`, `B_gpu`, `C_gpu`) выделяется память на GPU через `cuda.mem_alloc()`.
- **Копирование данных на GPU:** Матрицы `A` и `B` передаются на GPU с помощью `cuda.memcpy_htod()`.
- **Запуск CUDA-ядра:** Умножение выполняется в ядре `matrixMulKernel`. Каждый поток вычисляет один элемент результирующей матрицы `C`.
- **Размеры блока и сетки:** Используется блок 16×16 потоков (256 потоков на блок — типичное значение для GPU Tesla T4). Размер сетки вычисляется динамически под размер матриц.
- **Синхронизация:** `cuda.Context.synchronize()` гарантирует завершение всех потоков GPU до копирования результата.
- **Копирование результата:** Готовая матрица `C` возвращается на CPU через `cuda.memcpy_dtoh()`.

#### Объяснение ядра (`matrixMulKernel`)

```cuda
int row = blockIdx.y * blockDim.y + threadIdx.y;
int col = blockIdx.x * blockDim.x + threadIdx.x;
```

Каждый поток однозначно идентифицирует себя по индексам строки и столбца в результирующей матрице. Внутренний цикл по `k` вычисляет скалярное произведение строки `row` матрицы `A` и столбца `col` матрицы `B`.

**Что именно параллелизируется:** каждый из тысяч потоков GPU независимо вычисляет один элемент матрицы `C`, не требуя синхронизации с другими потоками. Это возможно потому, что элементы результирующей матрицы не зависят друг от друга, — задача является «смущающе параллельной» (embarrassingly parallel).

Граничная проверка `(row < rows_a && col < cols_b)` предотвращает обращение к памяти за пределами массивов, когда размер матрицы не кратен размеру блока.

---

## Почему GPU быстрее?

Умножение матриц имеет кубическую сложность O(N³), однако все элементы результирующей матрицы вычисляются **независимо** друг от друга. Это делает задачу идеальной для параллельных вычислений.

CPU выполняет операции последовательно (1–16 ядер), тогда как GPU Tesla T4 содержит 2560 CUDA-ядер, способных работать одновременно. При размере матрицы 2000×2000 требуется 4 000 000 независимых операций скалярного произведения — GPU запускает их параллельно, CPU — строго по одной.

## Результаты экспериментов

Все вычисления проводились в Google Colab на GPU **Tesla T4** (Driver 580.82.07, CUDA 12.8, pycuda 2026.1).

### Проверка корректности (2000×2000)

```
Соответствие результата с np.dot(): True   # GPU
Соответствие результата с np.dot(): True   # CPU
```

Оба метода дают результат, идентичный эталонному `np.dot()`.

### Вывод

С ростом размера матриц ускорение GPU монотонно увеличивается: от ~428× для матриц 100×100 до ~63415× для матриц 2000×2000. Это объясняется тем, что при малых матрицах накладные расходы на передачу данных и организацию потоков занимают бо́льшую долю времени, тогда как при больших размерах параллелизм GPU раскрывается полностью.
