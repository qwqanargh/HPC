# Сумма элементов вектора на CPU и GPU

Этот проект демонстрирует реализацию нахождения суммы элементов вектора двумя способами:
- **CPU** — последовательное суммирование в одном потоке.
- **GPU** — параллельная редукция с использованием CUDA.

---

## Описание функций

### `vector_sum_cpu(vector)`

Функция для последовательного сложения элементов вектора на **CPU**.

- **Параметры:** `vector` — входной вектор (numpy array, dtype=int32).
- **Возвращает:** `answer` — целочисленная сумма элементов.

#### Детали реализации

Суммирование выполняется в одном потоке с помощью цикла `for`: каждый элемент добавляется к накопленной сумме по очереди. Никакого параллелизма нет — при росте размера вектора время растёт линейно.

---

### `vector_sum_gpu(vector)`

Функция для параллельного сложения элементов вектора на **GPU** с использованием CUDA-ядра `addKernel`.

- **Параметры:** `vector` — входной вектор (numpy array, dtype=int32).
- **Возвращает:** `(answer, elapsed_seconds)` — результат и время выполнения (только вычислений на GPU, без учёта копирования данных).

#### Что конкретно распараллелено и почему

Сложение элементов вектора — **embarassingly parallel** задача: каждый элемент можно суммировать независимо от других. Это делает её идеальной для GPU.

Параллелизация реализована через алгоритм **параллельной редукции**:

1. **Загрузка данных.** Каждый CUDA-поток получает уникальный глобальный индекс (`blockIdx.x * blockDim.x + threadIdx.x`) и загружает один элемент вектора в `shared`-память своего блока. `Shared`-память на порядок быстрее глобальной, что критически важно для редукции.

2. **Редукция внутри блока.** На каждом шаге цикла `stride` уменьшается вдвое: активные потоки (с `threadIdx.x < stride`) суммируют пары элементов `sharedSum[i] += sharedSum[i + stride]`. За `log2(block_size)` шагов сумма всего блока концентрируется в `sharedSum[0]`. Барьер `__syncthreads()` гарантирует, что все потоки завершили шаг перед переходом к следующему.

3. **Запись частичных сумм.** Поток 0 каждого блока атомарно прибавляет `sharedSum[0]` к глобальному результату с помощью `atomicAdd`. Атомарная операция предотвращает гонку данных при одновременной записи из нескольких блоков.

#### Прочие детали реализации

- **Динамический shared memory.** Размер `shared`-буфера передаётся при запуске ядра (`shared=block_size * 4` байт), а не фиксируется в коде ядра. Это делает ядро корректным при любом размере блока.
- **Компиляция ядра вынесена** за пределы функции — один раз при импорте модуля, а не при каждом вызове.
- **`cuda.Context.synchronize()`** вызывается перед снятием временной метки, чтобы учесть реальное время выполнения GPU, а не момент постановки задачи в очередь.

---

### Почему GPU быстрее при больших векторах?

CPU суммирует элементы последовательно — одна операция за раз. GPU запускает тысячи потоков одновременно и выполняет редукцию за `log2(N)` синхронизированных шагов вместо `N` последовательных. Накладные расходы на запуск ядра практически постоянны, поэтому при малых векторах (1 000 элементов) они ещё сопоставимы с полезной работой и выигрыш минимален, а при больших — GPU значительно **выигрывает**.

## Результаты экспериментов

Все вычисления производились в Google Colab (GPU: Tesla T4, CUDA 12.2).

| Размер вектора | Время на CPU (сек) | Время на GPU (сек) | Ускорение (CPU / GPU) |
|:--------------:|:------------------:|:------------------:|:---------------------:|
| 1 000          | 0.000127           | 0.000096           | 1.31×                 |
| 50 000         | 0.006396           | 0.000047           | 135.48×               |
| 125 000        | 0.013334           | 0.000063           | 210.26×               |
| 250 000        | 0.058821           | 0.000106           | 556.92×               |
| 500 000        | 0.138406           | 0.000133           | 1 044.09×             |
| 1 000 000      | 0.241668           | 0.000202           | 1 196.73×             |

**Вывод:** даже при минимальном размере вектора (1 000 элементов) GPU уже быстрее CPU в ~1.3 раза. По мере роста размера ускорение резко возрастает: при 1 000 000 элементов GPU опережает CPU почти в **1 197 раз**. Такой высокий показатель достигается за счёт вынесения компиляции CUDA-ядра за пределы функции и корректного замера времени через `cuda.Context.synchronize()`.
